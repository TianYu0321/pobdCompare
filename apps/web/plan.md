# P3 UI 返工计划：游戏化双 BD 对比工作台

## 返工原因
当前 UI 像调试面板：装备卡片瀑布流、天赋散点图、中央空白、缺少游戏装备槽位感。

## 目标
暗黑游戏 UI 风格的双 BD 对比工作台，装备图标为视觉中心，少文字，点击/hover 展示详情。

## 阶段

### Stage 1: 基础设施（颜色+布局）
- [ ] 更新 tailwind.config.js 颜色（按文档指定色值）
- [ ] 重写 ComparePage 三栏布局（42/16/42）
- [ ] 单 BD 模式支持（轻量 dropzone）

### Stage 2: 左右 Build 面板
- [ ] 重写 BuildPanel（角色摘要 + 概览/装备/技能/天赋/计算 Tab）
- [ ] 概览 Tab：角色信息大数字展示
- [ ] 装备 Tab：固定装备槽位布局（游戏式网格）
- [ ] 技能 Tab：技能组卡片（带图标）
- [ ] 天赋 Tab：收益榜（下一点/路径包/移除损失），去掉散点树

### Stage 3: 中央 Diff Rail
- [ ] 分析前状态：等待 Build B / 点击开始对比
- [ ] 分析中状态：步骤进度条
- [ ] 分析后状态：DPS/一击线对比 + 关键差异 Top 3 + 攻击/防御切换
- [ ] 主技能选择交互

### Stage 4: 装备详情交互
- [ ] 点击装备槽打开详情抽屉
- [ ] 展示完整词条 + 与对方差异 + 替换收益（mock）
- [ ] 不兼容提示（非普通负收益）

### Stage 5: 构建验证
- [ ] npm run build
- [ ] 拖入测试数据验证

## 验收标准
1. 左右双输入框可用
2. 单 BD 模式不空白
3. 双 BD 对比模式
4. 中央 Diff Rail 有分析前/中/后状态
5. 装备区为固定装备槽位布局
6. 装备槽显示图标、装备名、DPS Δ、一击线 Δ
7. 点击装备槽可看详情
8. 技能 tab 展示技能组
9. 天赋 tab 展示收益榜
10. 攻击/防御切换
11. 不做评分、不做报告页主界面
